# 📝 Python 元组（Tuple）速查表

| 功能/操作      | 示例代码                                                     | 说明                             |
| -------------- | ------------------------------------------------------------ | -------------------------------- |
| **定义元组**   | `t1 = (35, 12, 98)`<br>`t2 = ('骆昊', 45, True, '成都')`     | 使用 `()` 定义，元素有序且不可变 |
| **空元组**     | `a = ()`                                                     | 空元组                           |
| **一元组**     | `b = ('hello',)`                                             | 只有一个元素时必须加逗号         |
| **类型查看**   | `type(t1)`                                                   | 返回 `<class 'tuple'>`           |
| **长度**       | `len(t1)`                                                    | 元素个数                         |
| **索引**       | `t1[0] → 35`<br>`t2[-1] → '成都'`                            | 支持正向、反向索引               |
| **切片**       | `t2[:2] → ('骆昊', 45)`<br>`t2[::2]`                         | 与列表切片规则相同               |
| **遍历**       | `for e in t1: print(e)`                                      | 可迭代                           |
| **成员运算**   | `12 in t1 → True`<br>`99 in t1 → False`                      | 判断元素是否存在                 |
| **拼接**       | `t3 = t1 + t2`                                               | 生成新元组                       |
| **比较**       | `t1 == t3`<br>`t1 <= (35, 11, 99)`                           | 按元素逐一比较                   |
| **打包**       | `a = 1, 10, 100`                                             | 自动打包为元组 `(1, 10, 100)`    |
| **解包**       | `i, j, k = a`                                                | 元组解包到多个变量               |
| **解包异常**   | `i, j = a` → `ValueError`                                    | 元素数与变量数不匹配时报错       |
| **星号解包**   | `i, *j, k = (1,10,100,1000)` → `i=1, j=[10,100], k=1000`     | 星号变量接收多个值，结果为列表   |
| **交换变量**   | `a, b = b, a`                                                | 直接交换，不需中间变量           |
| **多变量交换** | `a, b, c = b, c, a`                                          | 三个变量循环交换                 |
| **与列表比较** | - 元组不可变，列表可变<br>- 元组创建速度更快<br>- 更适合多线程环境 | 元组更轻量，列表更灵活           |
| **类型转换**   | `list((1,2,3)) → [1,2,3]`<br>`tuple([1,2,3]) → (1,2,3)`      | 列表与元组可互转                 |

------

📌 **总结要点**

- 元组是 **不可变序列**，适合存放固定数据。
- 与列表共享大部分操作（索引、切片、拼接、成员运算）。
- 常用于 **多变量赋值、函数返回多个值、数据保护**。
- 创建速度快，线程安全性更高。

------

# 📊 Python 列表（List） vs 元组（Tuple）对比速查表

| 特性           | 列表（List）                                                 | 元组（Tuple）                                                |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **语法**       | 方括号 `[]` 或 `list()`                                      | 圆括号 `()` 或 `tuple()`                                     |
| **可变性**     | ✅ 可变（支持增删改）                                         | ❌ 不可变（定义后不可修改）                                   |
| **方法数量**   | 丰富（`append`, `insert`, `remove`, `sort` 等）              | 仅 2 个：`count`, `index`                                    |
| **内存占用**   | 较大（为动态扩展预留空间）                                   | 较小（固定分配，更轻量）                                     |
| **创建速度**   | 较慢                                                         | 更快                                                         |
| **迭代速度**   | 基本一致                                                     | 基本一致                                                     |
| **适用场景**   | - 需要频繁修改数据<br>- 存储同类数据集合<br>- 使用列表推导式 | - 数据不可变时（保护数据）<br>- 作为字典键<br>- 函数返回多个值<br>- 存储异构数据（如坐标） |
| **单元素写法** | `[42]`                                                       | `(42,)` ← 必须加逗号                                         |
| **转换**       | `tuple([1,2,3]) → (1,2,3)`                                   | `list((1,2,3)) → [1,2,3]`                                    |

------

📌 **核心记忆口诀**

- **列表 = 可变的小胖子**（灵活但占空间）
- **元组 = 不可变的小瘦子**（轻量且安全）

# 🐍 Python 元组用法总结

## 1. 定义与特性

- 元组是有序序列，语法：`(x, y, z)`
- **不可变类型**：不能增删改元素
- 空元组：`()`
- 一元组：必须加逗号，如 `('hello',)`，否则只是普通类型

------

## 2. 基本操作

- 类型与长度

  ```python
  t = (35, 12, 98)
  print(type(t))   # <class 'tuple'>
  print(len(t))    # 3
  ```

- 索引与切片

  ```python
  t2 = ('骆昊', 45, True, '成都')
  print(t2[0])     # '骆昊'
  print(t2[-1])    # '成都'
  print(t2[:2])    # ('骆昊', 45)
  ```

- 成员运算

  ```python
  12 in t          # True
  'Hao' not in t2  # True
  ```

- 拼接与比较

  ```python
  t3 = t + t2
  print(t1 == t3)  # False
  ```

------

## 3. 打包与解包

- 打包

  ：多个值 → 元组

  ```python
  a = 1, 10, 100
  print(a)  # (1, 10, 100)
  ```

- 解包

  ：元组 → 多个变量

  ```python
  i, j, k = a
  print(i, j, k)  # 1 10 100
  ```

- 星号解包

  （接收多余元素，结果为列表）

  ```python
  a = (1, 10, 100, 1000)
  i, *j, k = a
  print(i, j, k)  # 1 [10, 100] 1000
  ```

------

## 4. 变量交换

- Python 支持直接交换变量值，无需中间变量：

  ```python
  a, b = b, a
  a, b, c = b, c, a
  ```

------

## 5. 与列表的比较

- **相同点**：都支持索引、切片、拼接、成员运算、遍历

- **不同点**：

  | 特性     | 列表 list | 元组 tuple   |
  | -------- | --------- | ------------ |
  | 可变性   | ✅ 可变    | ❌ 不可变     |
  | 线程安全 | 较差      | 更适合多线程 |
  | 创建速度 | 较慢      | 更快         |
  | 常见用途 | 动态数据  | 固定数据     |

- **性能对比**（创建相同元素序列）

  - 列表：约 0.635 秒
  - 元组：约 0.078 秒
     → 元组创建更快

------

## 6. 类型转换

```python
infos = ('骆昊', 43, True, '成都')
print(list(infos))   # 转换为列表

frts = ['apple', 'banana', 'orange']
print(tuple(frts))   # 转换为元组
```

------

## 7. 总结

- **列表**：可变，适合需要频繁修改的场景
- **元组**：不可变，适合固定数据、性能要求高、多线程环境
- 三者（列表、元组、字符串）本质上都是 **有序序列**，共享大部分操作方式
