# 此代码基于仓库内示例代码改进

## 玩法
玩家和电脑可以各自选择 3 名宝可梦（电脑随机选择）组成队伍，游戏开始时从 3 个宝可梦中选择一个出战，每个宝可梦拥有以下特性：
- HP：此宝可梦的血量，当宝可梦的剩余 HP 小于等于 0 时，该宝可梦会【昏厥】，需要更换其他可以行动的宝可梦
- 属性：宝可梦的属性会影响属性和抗性，相同属性的宝可梦会有一个相同的属性被动，宝可梦的属性会影响造成和受到的伤害
- 攻击力：决定其对其他宝可梦的伤害
- 防御力：来自其他宝可梦的伤害需要减去防御力的数值
- 闪避率：在战斗中成功躲闪开敌人攻击的概率

|属性|克制|被克制|属性被动|
|----|----|----|----|
|草|水|火|每回合回复 10% 最大 HP 值
|火|草|水|每次造成伤害，叠加 10% 攻击力，最多 4 层
|水|火|电|受到伤害时，有 50% 的几率减免 30% 的伤害
|电|水|草|当成功躲闪时，可以立即使用一次技能
|冰|水|火|对克制和被克制宝可梦造成220%和70%伤害

若无特殊说明，被克制的宝可梦受到来自克制的宝可梦的伤害翻倍，被克制的宝可梦对克制的宝可梦造成的伤害减半

## 优化：

### play.py

1. 选择宝可梦时深拷贝列表，新增team属性以便区分选择的宝可梦来自玩家还是电脑


### pokemon.py

1. 新增can_act属性，判断当前回合是否可以行动，当受到麻痹或者冰冻时，电脑失去本回合行动，玩家失去下一回合行动。
2. 重构add_status_effect(),当传入的 effect 携带要叠加的层数，仅将持续时间更新为两者最大值，不重复添加，若携带了层数，则持续时间更新为两者最大值，且将传入的层数叠加到已有状态的层数上。
3. 新增电属性宝可梦被动on_evade()，成功闪避一次攻击后，可以立即使用一次技能。


### skill.py

1.重构receive_damage()受伤行为，普通技能攻击伤害可被防御力减免，特殊伤害（烧伤等）无视防御，技能伤害改为基于当前实时攻击力计算。
2.为蓄力类技能新增蓄力条Gathering_strength。


### effects.py

1.新增层数属性，可叠加
2.烧伤造成固定伤害，中毒造成百分比伤害
