## 程序设计

本来代码只在给出的四个文件中添加,后面发现太乱了😓,改了文件结构

### 宝可梦

- 对于不同属性的宝可梦的的被动,直接覆写基类的方法,感觉这种实现方式不是特别完美
- 对于改变游戏流程的被动,如闪避后的额外回合,直接在主循环中实现

### 技能&状态效果

- 状态效果添加了清除函数,用于调整数值的改动,如减伤数值的调整
- 在实现迟一回合释放和麻痹,在pokemon基类里直接添加了cant_move,在主循环中检测是否可以行动
- 实现迟一回合释放也是做成了一种效果,在效果结束时再调用技能中的释放函数,感觉这种实现方式也不是很完美
- 减伤的技能释放时,状态效果立刻应用,这是写在技能中的,现在想来可以写在减伤效果的构造函数中

### 总结

总的来说对于oop的理念和实现不是特别熟,很多时候仅仅顾及眼前的功能实现,没有考虑以后的功能,没有完全按照oop的思想进行编写,(其实是因为偷懒了,懒得再定义方法).
