# 基础容器
## List（列表）：[]
一种有序、可变（可以修改）的数据集合。可以包含不同类型的元素（数字、字符串、甚至其他列表、字典等）。
### 常用
索引：my_list[index](从0开始，负数表示从末尾开始)

切片：my_list[start:end:step]获取子列表。

添加：append(item)(末尾添加), insert(index, item)(指定位置插入)

删除：remove(item)(删除第一个匹配项), pop([index])(删除并返回指定位置元素，默认末尾), del my_list[index]

修改：my_list[index] = new_value

长度：len(my_list)

遍历：for item in my_list: ...

## Dict（字典）： {}
一种无序、可变的数据集合。存储的是键值对（键和值相互对应）。键必须是不可变类型（如字符串、数字、元组），值可以是任意类型。字典通过键来快速查找对应的值。

### 部分常用
访问值：my_dict[key](如果键不存在会报错 KeyError), my_dict.get(key[, default])(安全访问，键不存在返回 None或指定的 default)

添加/修改：my_dict[key] = value(如果键存在则修改，不存在则添加)

删除：del my_dict[key], pop(key[, default])(删除并返回值)

检查键是否存在：key in my_dict

获取所有键：my_dict.keys()(返回视图)

获取所有值：my_dict.values()(返回视图)

获取所有键值对：my_dict.items()(返回视图，每个元素是 (key, value)元组)

遍历： for key in my_dict: ...(遍历键),
      for value in my_dict.values(): ..., 
      for key, value in my_dict.items(): ...

# 函数 def
## Lambda 匿名函数：
一种用一行代码定义简单函数的快捷方式。它没有名字（所以叫“匿名”），通常用于需要一个简单函数作为参数传递给其他函数的场景。

## Decorator 装饰器：
用一个函数（装饰器）去“包装”另一个函数（被装饰的函数）。在调用被装饰的函数时，实际上执行的是装饰器函数返回的那个“包装后”的函数。

# 面向对象
## Class（类）：
类是创建对象的蓝图或模板。它定义了对象将具有的属性（数据）和方法（操作数据的函数）。
### 相关概念
构造函数 __init__(self, ...)：在创建新对象时自动调用的特殊方法，用于初始化对象的属性。
self：类方法中指向当前对象实例本身的第一个参数（约定俗成叫 self）。
## Magic Methods（魔法方法）：
类中以双下划线开头和结尾的特殊方法（如 __init__, __str__, __len__）。解释器会在特定的操作或语法上下文中自动调用它们。
### 常见用法
__init__(self, ...)：对象初始化（构造）。

__str__(self)：当使用 print(obj)或 str(obj)时调用，返回对象的“用户友好”字符串表示。通常需要定义。

__repr__(self)：当在解释器中直接输入对象名或使用 repr(obj)时调用，返回对象的“官方”字符串表示（通常可用于重新创建该对象）。通常也需要定义。

__len__(self)：当使用 len(obj)时调用，返回对象的“长度”。

__getitem__(self, key)：当使用 obj[key]索引访问时调用。

__setitem__(self, key, value)：当使用 obj[key] = value索引赋值时调用。

__add__(self, other)：当使用 obj1 + obj2时调用。

__eq__(self, other)：当使用 obj1 == obj2时调用。

__call__(self, ...)：使对象可以像函数一样被调用 obj(...)。

## OOP 思想：
编程时，把一切看成一个个独立的“对象”（比如“学生”、“课程”）来设计和组织代码。
核心是：

### 封装：
把数据和操作数据的方法打包在一起，只留个接口，内部细节隐藏。

### 继承：
子类可以继承父类的特性，不用重复写代码。比如“电动车”继承“车”的所有特性，再加个“用电”的特性。

### 多态：
不同对象对同一指令可以做出不同反应。比如对“动物”下“叫”的命令，狗会“汪汪”，猫会“喵喵”。

# 文本处理
## re 正则表达式：
一种强大的“文本查找替换工具”。用一串特殊符号（规则）来描述你想要的文字模式（比如“所有以13开头的手机号”），然后计算机就能在文章里自动帮你找到或替换所有符合这个模式的内容。

# 代码格式
## 列表推导式：
一种简洁、优雅地创建新列表的语法结构。它基于现有的可迭代对象（如列表、元组、字符串、range等），通过一个表达式和一个可选的 for循环（以及可选的 if条件）来生成新列表。

## Type Hint（类型注释）：
给变量、函数参数和返回值贴上“类型标签”（比如 name: str）。它不影响程序运行，主要目的是让读代码的人（和智能编辑器）一眼就知道这里应该是什么类型，减少错误，更清晰。

# 进阶技巧
## generator 生成器（yield）：
一个“懒惰的”列表。允许你定义一个函数，这个函数能逐步产生（yield）一系列值，而不是一次性计算并返回所有值（像列表那样），它才现场算一个值给你。处理海量数据时能极大节省内存。用 yield关键字的函数就是生成器。
